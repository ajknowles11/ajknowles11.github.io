Games and Demos

Panarctica

Role: Lead Programmer (Gameplay, Animation, AI)

Developed as my second project working with Carnegie Mellon University's Game Creation Society, Panarctica is a vertical slice first-person shooter taking place in airships above a frozen world. This was my first time working with Unreal Engine (5.1), and I became comfortable working in a variety of areas.

Personal highlights include:

Automatic and semi-automatic hitscan weapons, with recoil, and animations and sounds for shooting and reloading. Uses both blueprint calls to play animations, and animation state machines.

Reactive AI that patrols initially, then becomes alert and chases the player on sight, shooting when within range. Enemies with shotguns are more direct and aggressive, while enemies with SMGs are more tactical and strafe around when shooting.

Some UI work, including a global settings menu for changing volume and mouse sensitivity, which persist between runs and are easy to change. Also, cutscenes are skippable to help encourage replayability.

Minion

Role: Programmer (Gameplay, Lighting/Line-of-sight)

My first project I worked on with CMU's GCS was Minion, an old-school 2D roguelike made in Unity. This was my first completed project in any game engine, and I learned many important skills including how to work effectively with git, and how to communicate between departments.

Personal highlights include:

Lighting algorithm integrated with player line-of-sight to create realistic shadows, and extensible generic light object with custom texture and color.

ASCII level editing using RexPaint to integrate light objects and obtainable items.

School Projects

MallocLab

C Dynamic Memory Allocator

The infamous assignment from CMU's 15-213 Intro to Computer Systems, MallocLab is a dynamic heap memory allocator designed to maximize throughput and minimize utilization, with efficient tracking of free blocks of variable size. It is a custom implementation of the C Malloc, Calloc, Realloc, and Free functions.

Features include:

Segregated list for free blocks: Different linked list for each size class, with sizes increasing in powers of 2 to make searching fast.

Footerless blocks when possible (meaning only headers containing size information), and coalescing of neighboring free blocks.

Github repository available upon request. Email ajknowles21@gmail.com.

ProxyLab

C HTTP Proxy Server, with Cache

Another assignment from CMU's 15-213 Intro to Computer Systems, ProxyLab is an HTTP Proxy server supporting GET requests and passing along custom headers to a host, and caching binary information before passing it back to the client.

Features include:

Multithreading to enable simultaneous requests, and simulatenous reads from the LRU cache for web objects.

Github repository available upon request. Email ajknowles21@gmail.com.

ShellLab

Tiny Unix Shell Written in C

Another assignment from CMU's 15-213 Intro to Computer Systems, ShellLab (aka TSHLab) is a small Unix shell implementation supporting both foreground and background jobs, as well as interrupt and stop signals from the user.

Features include:

Multiprocessing with signal handling to interrupt foreground processes and resume in the foreground or background, as well as reaping dead child processes.

Specify arg input and output files via '<' and '>'.

Github repository available upon request. Email ajknowles21@gmail.com.
